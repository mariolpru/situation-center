## Дизайн-система (сверено с Figma)

**Темы** — переключение дабл-кликом по логотипу; светлая по умолчанию; выбор в sessionStorage (`sc-theme`), новый заход всегда светлый. Тема = `data-theme="dark"` на `<html>`; переменные в tokens.css (`--hero-*`, `--btn-*`, `--c-red`).

- Светлая: красный `#BE171E`, текст `#09131c`/`#08131b`, hero светлая карта `moscow_light.png`
- Тёмная: красный `#EF3E4A`, hero тёмная карта `moscow.png`
- Кнопки светлой темы: default бордер 5px `#BE171E` + текст `#09131C`; hover — заливка `#BE171E` + белый текст. Радиус 48, паддинг 16/26, Golos Medium 17px
- Тёмно-синий «navy» `#161f26` (карточка «все события», кнопки этажей, заголовок панорамы)

**Шрифты (локальные, папка /font):** TT Bluescreens Trial (заголовки; СТАТИЧЕСКИЕ начертания — variable в Safari курсивит!) + Golos Text (текст). Заголовки: hero 120px, панорама/цифры 100px (`--fs-h1`), секции 54px (`--fs-h2`), лид/подзаголовки 29px, карточки/инпуты 22px, кнопки/даты 17px, текст статьи 20px. Межстрочный заголовков .887, трекинг −2% (у секционных 54px — трекинг 0).

**Инпуты:** фон `rgba(8,19,27,.08)`, радиус 20, паддинг 21/39, текст 22px. Карточка рассылки/модалки: `#ecedee` (сплошной!), радиус 30. Пагинация: ячейка 50×38 r20, активная — заливка `#BE171E`, hover — обводка 2px.

## Ключевые механики (js/main.js)

- **Лоадер**: пульс кругов лого, скрывается по load (или 4с страховка). `ready`-промис.
- **Печатающийся заголовок hero**: «Место,» печатается один раз; строки 2–3 перепечатываются каждые 4с (стирание снизу вверх, печать сверху вниз, красная каретка). PHRASES/ERASE_MS/TYPE_MS.
- **Ротация факта**: 3 цифры каждые 4с с count-up.
- **Точки на карте**: CSS-анимация transform/opacity (НЕ box-shadow — кляксы в Safari).
- **Панорама**: разбор здания по этажам (3 слоя floor-*.png), offset 300px, ±20% opacity, фейд у границ через mask; сцена растянута вверх до заголовка.
- **События ховер**: Material-ripple (2 красных круга из левого нижнего угла, стаггер), инлайн SVG.
- **Модалки**: делегирование `[data-open-form]` (экскурсия) и `[data-open-modal="id"]`; `.modal__state[data-state]` — мультисостояния; закрытие сбрасывает к первому состоянию. Фон блюрится (`body.form-open`).
- **Пагинация**: общий `setupPagination(el, total, onChange)` — окно `1 2 … c−1 [c] c+1 … 167 168`.
- **Мобильное меню**: строится из `.hero__menu a, .pagebar__menu a`.

## Правила работы

1. **Кеш-бастинг:** при ЛЮБОЙ правке css/js поднять `?v=N` во всех 5 html (сейчас v=15). Иначе браузеры держат старьё.
2. Деплой: commit → push → Pages пересобирается ~30-60с (проверка: `curl … | grep v=N`).
3. Коммиты от `mariolpru <mckrab@gmail.com>`, сообщения на русском.
4. Пуш в этот репо разрешён без подтверждений (создание НОВЫХ публичных сущностей — блокируется, просить юзера).
5. Line-art графика (карты, этажи) — квантованный PNG (Pillow quantize 256), НЕ lossy-webp (мылит линии). Фото — WebP q80.
6. Figma MCP: сначала `get_metadata` (структура), потом `get_design_context` на подузлы (большие узлы падают по токенам). Скриншоты браузер-панели запаздывают — проверять состояние через javascript_tool (но! панель блокирует программные inline-стили — hover проверять физическим hover-действием).
