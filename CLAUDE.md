# Ситуационный центр — прототип сайта

HTML-прототип лендинга «Ситуационный центр градостроительного комплекса Москвы» для показа клиенту.
Статичный сайт: HTML/CSS/JS без сборки, анимации — Motion (motion.dev) через ESM-CDN.

## Ссылки

- **Прод:** https://mariolpru.github.io/situation-center/ (GitHub Pages, репо `mariolpru/situation-center`, ветка main, корень)
- **Локально:** `python3 -m http.server 5599` в корне (ES-модули требуют http). Обычно уже запущен.

## Структура

```
index.html        — главная (hero-карта, панорама этажей, события, новости, рассылка)
news.html         — детальная новость (статья + шеринг)
event.html        — детальное событие (статья + «О спикере» + «Детали» + попап лекции)
news-list.html    — «Все новости» (список 10 + пагинация)
events-list.html  — «Все события» (сетка 18 ev-card + пагинация)
css/fonts.css     — @font-face локальных шрифтов (папка /font)
css/tokens.css    — переменные: цвета, 2 темы, типографика, отступы
css/base.css      — reset, .container, .btn, .eyebrow, .arrow-circle
css/components.css— все блоки + страницы (найдёшь по комментариям-секциям)
css/layout.css    — адаптив (1279/1024/767/600/380) + мобильное меню
js/main.js        — все модули (IIFE с guard'ами по наличию элементов)
assets/img, assets/icons, font/
```

## Правила работы

1. **Кеш-бастинг:** при ЛЮБОЙ правке css/js поднять `?v=N` во всех 5 html (сейчас **v=18**). Иначе браузеры держат старьё.
2. Деплой: commit → push → Pages пересобирается ~30-60с (проверка: `curl … | grep v=N`).
3. Коммиты от `mariolpru <mckrab@gmail.com>`, сообщения на русском.
4. Пуш в этот репо разрешён без подтверждений (создание НОВЫХ публичных сущностей — блокируется, просить юзера).
5. Line-art графика (карты, этажи) — квантованный PNG (Pillow quantize 256), НЕ lossy-webp (мылит линии). Фото — WebP q80.
